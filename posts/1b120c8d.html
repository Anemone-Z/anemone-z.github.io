Obsidian Plugin - Annotator

发表于2024-05-30|更新于2024-06-09|ObsidianPlugin
|字数总计:751|阅读时长:2分钟|阅读量:

https://github.com/elias-sundqvist/obsidian-annotator/blob/master/README.md の 翻译。

简介

https://github.com/elias-sundqvist/obsidian-annotator

这是 Obsidian 的一个插件。它允许您打开和注释 PDF 和 EPUB 文件。

该插件基于 https://web.hypothes.is/ ，但经过修改，将注释存储在本地 markdown 文件中，而不是互联网上。

已知的问题

插件在 iOS 16.3 或更高版本上不起作用。请关注 #289 问题状态以了解最新情况。

如果注释在不同平台上发生了更改，则阅读器不会显示注释。

入门

将属性添加 annotation-target 到 Obsidian 笔记的正文中，其值与 EPUB/PDF 文件的位置相对应。该位置可以是保管库中的文件，也可以是在线文件。

然后，您可以在打开的注释窗格中选择“更多选项”（右上角的三个点），然后会出现一个新选项“注释”。

该插件会自动尝试根据文件路径确定文件是否为 epub 或 pdf ，但如果这不起作用，您也可以添加属性 annotation-target-type 并手动指定它是否是 epub 或 pdf 。

如果您安装了 dataview ，那么您也可以使用 dataview 属性指定注释目标。在这种情况下，可以使用黑曜石风格的链接代替纯文本路径。

警告！不要重命名原始 pdf 或 epub 文件！在这种情况下，插件将丢失注释和文件之间的联系。

注释

注释一目了然。使用鼠标选择文本即可开始。

将来，添加彩色高亮和图像/区域高亮会很好。不过，这些功能必须先添加到 hypothes.is 中。请参阅以下相关问题： hypothesis/product-backlog#198 、 hypothesis/product-backlog#669 。

Markdown 中的注释

要返回常规的 Obsidian Markdown 编辑模式，您可以选择 more options → Open as MD 。每个注释都有一个带有块引用的引用块。修改这些块时要小心。对 PREFIX、HIGHLIGHT 和 POSTFIX 进行少量编辑通常没问题。但如果编辑太大，hypothesis 可能无法再识别相应的文本。

COMMENT 区域可以自由编辑。（但要确保它仍然是引用块的一部分。）

TAGS 区域应由以逗号分隔的黑曜石标签列表组成。（如 #tag1, #tag2 , #tag3 ）

暗黑模式

该插件内置暗黑模式支持。要切换暗黑模式，请在注释时选择 more options → Toggle Dark Mode 。您还可以在插件的设置选项卡中调整暗黑模式行为。

链接到注释

单击注释块引用的黑曜石链接时，将打开相应的文件并滚动到相关的突出显示。如果文件已在窗格中打开，则链接将导致现有窗格滚动。
