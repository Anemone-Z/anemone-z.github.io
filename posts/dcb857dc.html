Obsidian Plugin - Paste Image Rename

发表于2024-04-28|更新于2024-06-09|ObsidianPlugin
|字数总计:1.7k|阅读时长:5分钟|阅读量:

简介

📢 从 1.4.0 开始，粘贴图片重命名成为通用重命名插件 可以处理添加到 Vault 的所有附件。

Paste Image Rename 的灵感来自 Zettlr ， Zettlr 显示了一个提示，允许用户重命名图像，如果您希望您的图像被清晰地命名和组织，这是一个很大的帮助。

粘贴图片重命名插件不仅实现了 Zettlr 的功能，还允许您自定义图片名称的生成方式，最终通过根据规则自动重命名图片来让您摆脱麻烦。

用法

安装插件后，您只需将图像粘贴到任何文档中，重命名提示将显示：

通过键入新名称并单击 Rename （或仅按回车键），图像将被重命名，内部链接将被替换为新名称。

如果将 Image name pattern 设置为 {{fileName}} （这是 1.2.0 之后的默认行为）， New name 将作为活动文件的名称生成。

设置 frontmatter

在向一个文档添加大量图像时，人们可能希望图像以相同的格式命名，这就是有用的地方。

首先在 frontmatter 设置 imageNameKey 的值：

然后粘贴一张图片，你会注意到 New name 已经生成为 my-blog ，这正是 imageNameKey 的值。

您可以通过更新设置中的 Image name pattern 的值来更改生成新名称的模式。

有关详细说明和其他功能，例如自动重命名，请参阅设置。

为重复的名称添加前缀/后缀

如果有同名文件，插件将始终尝试添加前缀/后缀。

让我们从上一节继续并粘贴第二张图片，提示符仍将显示新名称为 my-blog ，现在如果我们只需单击 Rename ，文件将重命名为 my-blog-1.png ，而不是 my-blog.png ：

后缀是根据默认设置生成的：-1

  • 由于 Duplicate number at start 为 false ，因此使用后缀而不是前缀。
  • Duplicate number delimiter 放在数字 -1 之前。

如果我们粘贴第三张图片而不编辑 New name 输入，它的名称将是 my-blog-2.png ，这个数字会根据 my-blog 的最大数量增加-?.png“。

如果您在设置中启用 Auto rename ，则此功能特别强大，您可以不假思索地添加新图像，它们将按模式和 imageNameKey 设置依次重命名。

批量重命名过程

1.3.0 中的新增功能。

您可以使用命令 Batch rename embeded files in the current file 重命名当前文件中的图像和其他附件（甚至注释）。

上图演示了如何通过此功能重命名所有 PNG 文件。

您也可以将图像重命名为相同的名称，并让插件处理名称重复数据删除。在此处观看视频演示：https://i.imgur.com/6UICugX.mp4

立即批量重命名所有图像

1.5.0 中的新功能。

命令 Batch rename all images instantly (in the current file) 将根据设置中的 Image name pattern 重命名当前文件中的所有图像。

这是使用带有某些参数的 Batch renaming process 的快捷方式， 使日常图像重命名变得更加容易。

请注意，此命令没有确认，请谨慎使用！

处理所有附件

1.4.0 中的新功能。

粘贴图像重命名不仅仅是粘贴图像的插件，它还具有潜力处理添加到 Vault 的所有附件，无论它们是否粘贴或拖拽。

要使用此功能，您需要在设置中启用 Handle all attachments 选项。

此外，您可以配置 Exclude extension pattern 以忽略文件与给定的扩展模式匹配。

常见问题

  • 问：我粘贴了一张图片，但没有显示重命名提示。
  • 答: 这可能是因为您正在使用 Windows 系统并从文件粘贴(即图像是从文件资源管理器复制的，而不是从浏览器或图像查看器)。在 Windows 中，从一个文件粘贴就像一个常规的文件传输，原始文件名被保留，而不是由黑曜石创建并命名为 Pasted image ... 。您需要打开 Handle all attachments 的设置，使其在这种情况下工作。

配置

Image name pattern

图像名称模式。该模式指示应该如何生成新名称。

可用变量:

  • {{fileName}}: 活动文件的名称，没有 .md 扩展名。
  • {{imageNameKey}}: 该变量从标记文件的前端读取，从相同的键 imageNameKey 读取。
  • {{DATE:$FORMAT}}: 使用 SFORAT 格式化当前日期， $FORAT 必须是 Moment.js 格式化字符串，例如 {{DATE:YYYY-MM-DD}} 。

下面是一些从模式到图像名称（按顺序重复）的例子。变量: fileName="My note"，imageNameKey="foo"：

  • {{fileName}}:My note,My note-1,My note-2
  • {{imageNameKey}}:foo,foo-1,foo-2
  • {{imageNameKey}}-{{DATE:YYYYMMDD}}:foo-20220408,foo-20220408-1,foo-20220408-2

Duplicate number at start (or end)

开头（或结尾）的重复数字。如果启用，重复数字将在开始时作为图像名称的前缀添加，否则将在结束时作为图像名称的后缀添加。

Duplicate number delimiter

重复数字分隔符。为重复名称生成数字前缀/后缀的分隔符。例如，如果值为 - ，后缀类似于 -1 、 -2 、 -3 ，前缀类似于 1- 、 2- 、 3- 。只允许在文件名中有效的字符。

Always add duplicate number

总是添加重复数字。如果启用，重复的数字将始终添加到图像名称。否则，只有在重复名称时才会添加它。

Auto rename

自动重命名。默认情况下，重命名模式将始终显示为在重命名之前进行确认，如果设置了此选项，则图像将在粘贴后自动重命名。

Handle all attachments

处理所有附件。默认情况下，这个插件只处理以 Pasted image 开头的图像，这是 Obsidian 用来从粘贴内容创建图像的前缀。如果设置了这个选项，这个插件将处理在保险库中创建的所有附件。

Exclude extension pattern

排除扩展模式。此选项只有在启用 Handle all attachments 时才有用。编写一个正则表达式模式来排除被处理的某些扩展。只使用第一行。

Disable rename notice

禁用重命名通知。如果您不希望在重命名图像时看到通知，请关闭此选项。注意，当链接发生更改时，Obsidian 可能会显示一个通知，此选项不能禁用该通知。
